# A10 Core Foundation — Framework Visual Orientation v0.2.0

This folder contains a browser-only framework visual orientation page for the `a10-core-foundation` repository.

It is intended for technically interested readers who want a compact visual map of A10 before reading the core design note and claim-boundary documents.

## Open

```text
docs/framework_visual_orientation/index.html
```

No Python, Streamlit, external API, or internet connection is required.

## Purpose

The page explains:

- A10 as a structured-prior / geometric-steering methodology,
- the repeated A10 loop:
  mission variable → bottlenecks/failure boundaries → reduced surrogate → structured prior/gate/barrier → robust evaluation → claim boundary,
- how the core foundation differs from domain-specific A10 repositories,
- why domain validation does not transfer automatically,
- the claim-boundary discipline.

## Non-claims

This page does not claim that:

- A10 is a proven fundamental law,
- A10 is a universal theory of nature,
- A10 applications are automatically valid,
- any industrial or safety validation has been completed,
- visual orientation replaces expert review or domain-specific evidence.
